# Pnpm Migration Summary

## Overview

This document describes the complete migration of this monorepo from Bun-based package management workflows to pnpm.

## Changes Implemented

### Workspace and Package Management

- Added `pnpm-workspace.yaml` to define monorepo workspace packages.
- Added `packageManager: pnpm@10.18.0` to the root and workspace package manifests.
- Removed Bun lockfile and prepared the repository for pnpm lockfile generation during the first successful `pnpm install`.

### Dependency and Script Updates

- Removed Bun-specific dependencies such as `@types/bun` and the runtime `bun` package where it was only supporting Bun-specific tooling.
- Updated scripts across packages from Bun execution patterns to pnpm-compatible commands.
- Replaced Bun test runner usage with Vitest in relevant workspaces.
- Updated TypeScript runtime scripts to use `tsx` where required.

### Runtime Compatibility

- Replaced Bun-specific server bootstrap in `apps/http-server` with a Node-compatible Hono server adapter.

### Task and Deployment Automation

- Updated Taskfile commands from Bun/Bunx to pnpm and pnpm dlx equivalents.
- Updated deployment and setup documentation to reference pnpm commands.

### Documentation Alignment

- Updated core repository documentation and internal docs to describe pnpm-based setup and workflows.

## Validation Guidance

After pulling this change, run the following:

```bash
pnpm install
pnpm -r run type-check
pnpm -r run lint
pnpm -r run test
```

## Notes

- Archived third-party Bun reference documentation under `docs/external-documentation` remains intact as external reference material and is not used for active project workflows.
